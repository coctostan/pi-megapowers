## Goal

Complete the migration to individual task files (`tasks/task-001.md` etc.) as the sole runtime source of truth by removing all runtime dependencies on the monolithic `plan.md`. The pipeline tool will read task content directly from task files, and the planâ†’implement gate will check for task file existence instead of `plan.md`.

## Acceptance Criteria

1. `pipeline-tool.ts` reads the current task's content via `readPlanTask(cwd, slug, taskIndex)` instead of reading `plan.md` and calling `extractTaskSection()`
2. The `planSection` passed to `runPipeline` is the task file's `.content` (markdown body after frontmatter)
3. When `readPlanTask` returns `undefined` for the requested task index, the pipeline tool returns an error message referencing task files (not `plan.md`)
4. A new `RequireTaskFilesGate` type exists in `types.ts` with `{ type: "requireTaskFiles" }`
5. `gate-evaluator.ts` evaluates `requireTaskFiles` gates as passing when `listPlanTasks(cwd, slug).length > 0`
6. `gate-evaluator.ts` evaluates `requireTaskFiles` gates as failing (with descriptive message) when no task files exist
7. `feature.ts` planâ†’implement transition uses `{ type: "requireTaskFiles" }` instead of `{ type: "requireArtifact", file: "plan.md" }`
8. `bugfix.ts` planâ†’implement transition uses `{ type: "requireTaskFiles" }` instead of `{ type: "requireArtifact", file: "plan.md" }`
9. `tool-signal.ts` error message for "no tasks found" references task files, not `plan.md`
10. `task-deps.ts` error message for "no tasks found" references task files, not `plan.md`
11. `plan.md` is still generated by `legacy-plan-bridge.ts` during plan approval (no change to generation)

## Out of Scope

- Removing `plan.md` generation or `legacy-plan-bridge.ts`
- Removing `extractTaskSection` or `plan-parser.ts` utilities (may have other callers or future use)
- Changing the plan authoring flow (`megapowers_plan_task` / `megapowers_plan_review`)
- Changing the `deriveTasks()` fallback path that parses `plan.md` for legacy plans
- Migrating existing completed plans to the new format
