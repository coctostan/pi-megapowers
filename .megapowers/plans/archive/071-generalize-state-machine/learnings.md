- The `diagnose` phase has a naming mismatch: the phase is named `diagnose` but its artifact is `diagnosis.md` — `deriveToolInstructions` must derive the save-phase name from the artifact filename base (strip `.md`), not the phase name, or the LLM gets instructed to call `megapowers_save_artifact({ phase: "diagnose" })` which the system won't recognize.
- Bugfix `reproduce` and `diagnose` are NOT blocking phases: despite being analogous to feature's `brainstorm`/`spec`, the existing write policy never blocked source writes during these phases — omitting `blocking: true` from their config is intentional behavioral equivalence, not an oversight.
- `brainstorm` is open-ended but still produces an artifact: it needs `artifact: "brainstorm.md"` explicitly in the config so that `brainstorm_content` gets populated in prompt injection — open-ended doesn't mean no artifact.
- Default-throw in a gate evaluator switch is the right TDD scaffold: starting `evaluateGate` with `default: throw new Error(\`Unknown gate type: ...\`)` ensures each new gate type has a proper red-green cycle; a default no-op return would make every unimplemented gate silently pass.
- Config validation at module level (not inside `getWorkflowConfig`) catches malformed configs at import time: running `validateWorkflowConfig()` before building the registry means a broken workflow config surfaces as a module load error rather than a silent runtime bug on first use.
